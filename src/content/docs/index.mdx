## Introduction

Thornberry is a modern, accessible component library built on top of [Ark UI](https://ark-ui.com/) and styled with [Tailwind CSS](https://tailwindcss.com/). It implements the Omni design system and provides a comprehensive set of reusable components for building web applications.

### Key Features

- **Accessibility First**: Built on Ark UI, ensuring ARIA-compliant, keyboard-navigable components
- **Fully Typed**: Written in TypeScript with comprehensive type definitions
- **Customizable**: Leverages Tailwind CSS and class-variance-authority for flexible styling
- **Copy & Paste**: Components are designed to be copied into your project, not installed as a dependency
- **Framework**: Built with Next.js 16 and React 19

## Getting Started

Thornberry uses the [shadcn](https://ui.shadcn.com/) registry system, allowing you to add components to your project on-demand.

### Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js**: Version 18 or higher
- **Bun** or **npm/pnpm/yarn**: Package manager of your choice
- **React**: Version 19 or higher
- **Next.js**: Version 16 or higher (if using Next.js)
- **Tailwind CSS**: Version 4 or higher

### Installation

1. **Install Tailwind CSS** in your project if you haven't already:

```bash
npm install tailwindcss @tailwindcss/postcss
```

2. **Install shadcn CLI** to manage components:

```bash
npx shadcn@latest init
```

3. **Configure your project** to use the Thornberry registry. Update your `components.json`:

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "default",
  "tailwind": {
    "config": "tailwind.config.ts",
    "css": "src/app/globals.css",
    "baseColor": "neutral",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui"
  }
}
```

4. **Add components** to your project using the shadcn CLI:

```bash
npx shadcn@latest add button
```

### Quick Start Example

After adding a component, you can import and use it in your application:

```tsx
import { Button } from "@/components/ui/button";

export default function Home() {
  return (
    <div>
      <Button variant="solid">Click me</Button>
    </div>
  );
}
```

## Requirements

### Runtime Dependencies

Thornberry components require the following packages:

- **@ark-ui/react** (^5.15.1): Headless UI components
- **class-variance-authority** (^0.7.1): CVA for variant-based styling
- **clsx** (^2.1.1): Conditional className utility
- **tailwind-merge** (^3.3.1): Merge Tailwind classes without conflicts
- **lucide-react** (^0.553.0): Icon library (optional, for icon components)

### Peer Dependencies

- **react** (^19.1.0)
- **react-dom** (^19.1.0)
- **tailwindcss** (^4.1.7)

These dependencies are automatically installed when you add components through the shadcn CLI.

### Customizing Theme

To customize the theme, modify the CSS variables in your `globals.css`. You can adjust colors, border radius, and other design tokens to match your brand.

### Dark Mode

Thornberry supports dark mode out of the box. Use the `.dark` class on a parent element to enable dark mode:

```tsx
<html className="dark">
  {/* Your app */}
</html>
```

Or use a theme provider like `next-themes` for dynamic theme switching.

## Typography

Thornberry doesn't enforce a specific typography system, giving you the flexibility to use any font family and scale that fits your design.

### Recommended Setup

For consistency with the Omni design system, consider using a modern sans-serif font stack:

```css
@layer base {
  body {
    @apply font-sans antialiased;
  }
}
```

### Font Configuration

Configure your fonts in `tailwind.config.ts`:

```ts
import type { Config } from "tailwindcss";

export default {
  theme: {
    extend: {
      fontFamily: {
        sans: ["var(--font-geist-sans)", "system-ui", "sans-serif"],
        mono: ["var(--font-geist-mono)", "monospace"],
      },
    },
  },
} satisfies Config;
```

### Using Next.js Fonts

If you're using Next.js, leverage the built-in font optimization:

```tsx
import { GeistSans } from "geist/font/sans";
import { GeistMono } from "geist/font/mono";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={`${GeistSans.variable} ${GeistMono.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

## Component Architecture

### Design Principles

Thornberry components follow these core principles:

1. **Composition**: Components are built from smaller, composable primitives
2. **Flexibility**: Use props and variants to customize appearance and behavior
3. **Accessibility**: ARIA attributes and keyboard navigation built-in
4. **Type Safety**: Full TypeScript support with exported prop types

### Component Structure

Most components follow this pattern:

- **Root Component**: The main wrapper component
- **Sub-components**: Composable parts (e.g., `AccordionItem`, `AccordionItemTrigger`)
- **Props Interface**: Exported TypeScript interface for all props
- **Variants**: Styling variants using class-variance-authority

### Usage Pattern

```tsx
import { ComponentRoot, ComponentItem } from "@/components/ui/component";

<ComponentRoot>
  <ComponentItem>
    {/* Content */}
  </ComponentItem>
</ComponentRoot>
```

## Utilities

### cn() Helper

Thornberry includes a `cn()` utility function for merging Tailwind classes:

```ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

This utility combines `clsx` for conditional classes and `tailwind-merge` to prevent class conflicts.

### Usage

```tsx
import { cn } from "@/lib/utils";

<div className={cn("base-class", isActive && "active-class", className)} />
```
